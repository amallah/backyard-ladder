## Codebase Patterns
- Supabase client is exported from `src/lib/supabase.ts` as `supabase` (singleton)
- Use `supabase.from("table").select().maybeSingle()` for optional single-row queries (returns null if not found, no error)
- Project uses Next.js App Router with TypeScript; typecheck via `npm run typecheck`
- All lib utilities go in `src/lib/`
- Dynamic pages use `params: Promise<{ slug: string }>` and `await params` (Next.js 16 App Router pattern)
- Split server components (data fetching + notFound) from client components (interactive / third-party React libs like react-qr-code)
- Use `headers()` from `next/headers` to get request host for building absolute URLs in server components
